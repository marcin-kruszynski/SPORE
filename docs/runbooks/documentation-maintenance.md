# Documentation Maintenance Runbook

## Rules

- Every significant change updates at least one canonical doc.
- New docs must be indexed.
- Research, decisions, and plans stay separated by folder responsibility.
- Prefer canonical updates over note sprawl.

## Hygiene Checklist

- `docs/INDEX.md` updated
- `docs/index/DOCS_INDEX.md` updated
- `docs/index/docs_manifest.yaml` updated
- `README.md`, `AGENTS.md`, and `docs/runbooks/local-dev.md` updated when environment or runtime assumptions change
- related docs linked
- ownership/status metadata present where relevant
