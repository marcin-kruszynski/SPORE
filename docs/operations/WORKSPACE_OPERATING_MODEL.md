# Workspace Operating Model

## Responsibility Split

- docs: architecture, governance, and decisions
- config/workspace: declarative composition artifacts
- schemas: contracts for future validators/services
- packages/apps/services: module ownership and boundaries

## Ownership Defaults

- bootstrap-agent: structure and governance seeds
- future-architect: architecture evolution
- future-orchestrator: runtime and execution flows
- doc-steward: docs hygiene and index integrity
