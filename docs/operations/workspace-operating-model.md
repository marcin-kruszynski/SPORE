# Workspace Operating Model

SPORE treats the repository as a controlled design workspace.

## Core Expectations

- Documentation is mandatory, not optional.
- Configuration examples and templates should evolve together with architecture notes.
- Research, decisions, and plans are separate artifact types.
- References stay external in identity, even when mirrored locally.

## Working Loop

1. Research the question.
2. Update the relevant canonical documents.
3. Update config, templates, or schemas if the docs imply a structural change.
4. Record decisions or open questions.
5. Only then begin implementation work in the relevant package or service.
