# Observability Model

## Core Signals

- state transitions by session and workflow step,
- queue depth / active worker counts,
- handoff latency between roles,
- test/review outcomes,
- docs update compliance markers.

## Operator Surfaces

- terminal feed for fast operational triage,
- web activity stream for broad visibility,
- structured event archive for audit.

## Policy

Observability must be useful before scale optimization. Instrumentation and lifecycle semantics come before throughput tuning.
