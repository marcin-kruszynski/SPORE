# Comparative Analysis

## Overstory

Strengths:
- explicit hierarchy and delegation,
- disciplined role separation,
- documentation and operational rigor.

Borrow:
- orchestrator -> lead -> worker planning model,
- clear scoped role boundaries and handoff discipline.

## Gastown

Strengths:
- operator-centric session workflows,
- persistent live process supervision,
- practical terminal ergonomics.

Borrow:
- durable session mindset,
- activity feed and inspectability-first operations.

## Mulch

Strengths:
- learning accumulation and project memory patterns.

Borrow:
- explicit learning capture separate from execution control.

## Beads

Strengths:
- structured task/dependency state and durable planning context.

Borrow:
- dependency-aware planning and state tracing model.

## PI Mono

Strengths:
- modular runtime and UI package ecosystem,
- extension-oriented agent tooling surfaces.

Borrow:
- PI-first runtime integration path and package boundaries.

## SPORE Distinction

SPORE combines these ideas into a configuration-first, docs-first orchestration foundation that is explicitly project/domain/profile-aware and not a clone of any single reference.
