## Discipline Guidance for SPORE

- Keep explicit plan-build-review loops and avoid implicit execution drift.
- Treat context and documentation as code-like assets with ownership and update discipline.
- Define tool boundaries and restrictions before broad autonomous execution.
- Introduce multi-agent collaboration through explicit contracts, not ad-hoc role behavior.
- Build production readiness through governance, observability, and evaluation checkpoints.

## Bootstrap Application

These points justify SPORE's docs-first operating model, ADR discipline, phased roadmap, and strict scope boundary against premature runtime implementation.
