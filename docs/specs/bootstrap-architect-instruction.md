# Bootstrap Architect Merged Instruction

This document merges the two source prompts into one execution contract for SPORE bootstrap.

## Mission

Build the development environment and project foundation for a future modular agent-swarm management system. Do not build the final platform yet.

## Combined Priorities

1. Clean repository structure.
2. Documentation-first knowledge system.
3. Reference study area with upstream repositories.
4. Configuration model for agents/profiles/teams/workflows/projects/domains.
5. Planning and governance system.
6. Local docs embeddings/search scaffold.
7. Future readiness for CLI, TUI, Web UI, and live sessions.
8. PI-first runtime integration strategy.

## Non-Negotiable Constraints

- documentation before implementation,
- modular and declarative composition,
- explicit boundaries between knowledge, configuration, orchestration, and interfaces,
- reference concepts without cloning implementation,
- incremental and auditable execution.

## Required Artifacts

- docs index and manifest,
- architecture and governance docs,
- ADRs,
- roadmap with phases and waves,
- config/workspace/schema scaffolds,
- references manifest and synthesis,
- tools/docsearch contract and usage plans,
- bootstrap status and completion summary.
